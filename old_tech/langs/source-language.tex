Most of the syntax is more or less standard.
Qualifiers $\src{q}$ can be attached to pointers to signal ownership.
Hereby, the qualifier $\src{\full}$ means "fully owned", meaning we may and also must delete the pointer at some point, while we cannot do the same for $\src{\half}$, which forbids us to delete.
Poison $\src{\poison}$ marks locations in the execution-context $\src{\Omega}$ as ,,to-be-deleted'' ($\src{\poisonless}$) or ,,deleted'' ($\src{\poisoned}$).
$\src{\commlib}$ contains a list of identifiers referencing what functions are considered a component.
Anything not present in $\src{\commlib}$ is considered to be a context.
We describe the act of calling or returning from component to context or vice versa as ,,crossing the boundary''.
The heap is split into two parts, such that any component listed in $\src{\commlib}$ will allocate in $\src{H^{\comp}}$ and anything else in $\src{H^{\ctx}}$.
Objects pointed at by a location $\src{\loc}$ are marshalled when they are passed across the boundary.
Without the heap sandboxing, contexts can arbitrarily rewrite a component's memory, which incurs compiler correctness issues.
We chose to distinguish between successful final results ($\src{\finalexprnoerr}$) and potentially crashed results ($\src{\finalexpr}$).
Note that a final result may also be an identifier, which we keep around as abstract representation for locations of pointers.
However, we still distinct them from ordinary values ($\src{\valueexpr}$), since they don't behave like normal values.
For example, we cannot make canonical typing lemmas for them without the additional information contained in a non-empty typing-context.
When discussing secure compilation, there is a notion of ,,context'' and ,,component'', where the latter is usually the part one cares about.
We use the term ,,context-switching'' with its usual meaning, i.e. change in control flow to some other procedure.
For $\src{\comm}$, the representation $\src{\ctxtocomp}$ signalizes a context switch from context to component, $\src{\comptoctx}$ from component to context, and $\src{\nocomm}$ signals an internal change, either inside the context or inside the component.
The state $\configs$ carries information on what the current context is and the continuation stack $\src{\kontstack}$ is used in the semantics to mark events with above policy accordingly.
Continuations $\src{K}$ in the stack are annotated with the name of the function $\src{foo}$ the continuation originates from.
Programs $\src{prog\ \library_{ctx}\ \library_{comp}}$ contain two lists of top-level definitions $\src{\library_{ctx}}$ and $\src{\library_{comp}}$.
$\src{\library_{ctx}}$ takes the role as attacker code.
We reason explicitly about substitutions in the dynamic semantics.

Evaluation is mostly straightforward, the only interesting cases involve the pointers.
Specifically, \Cref{tr:w-e-delete} demonstrates why we need the poison-tag on the locations: Regardless of the current tag, emit a $\src{Dealloc\ \loc}$ event and mark $\src{\loc}$ as poisoned ($\src{\poisoned}$).
The intuitive solution, removing the mapping on deletion, doesn't allow to run programs that delete twice.
However, we want to model memory effects and show that such situations never happen, even though they {\em could}.
Given a poisoned location, we can still do everything with it: reading, writing, or deletion.
When generating the static context from the execution context, \Cref{tr:t-poison-store} allows us to disregard deleted locations which will help us reason that the given execution could not have been happening if the program was well-typed to begin with.
For sake of readablity, we will omit $\src{\gamma}=\src{\hole{\cdot}}$ when writing down evaluation steps.
That is, whenever we have $\exprevals{\configs}{e\hole{\cdot}}{\configs'}{e'\hole{\cdot}}{\event}$, it's written $\exprevals{\configs}{e}{\configs'}{e'}{\event}$.
Moreover, $\src{\gamma}$ as postfix binds anything before it up to $\triangleright$, so $\src{let\ x=e_{1}\ in\ e_{2}\gamma}$ is $\src{(let\ x=e_{1}\ in\ e_{2})\gamma}$.

      \typechecks{x:\nat,\hole{\cdot}\circ x:\nat,\hole{\cdot}}{let\ z=new\ x\ in\ let\ w = z[1337]\ in\ delete\ z; w}{\nat}
    }}
  \bigskip

  \bigskip
  \noindent\makebox[\textwidth]{%
    \typerulederiv{$\text{\Cref{tr:t-get}}$}{
      \typerulederiv{$\text{\Cref{tr:t-var}}$}{
        \src{\wptr}\not=\src{\ptr}&
        \src{\nat}\not=\src{\ptr}
      }{
        \typechecks{\Gamma'}{z}{\wptr}
      } &
      \typerulederiv{$\text{\Cref{tr:t-var}}$}{

        \src{\nat}\not=\src{\ptr}

        \typechecks{\Gamma'}{x}{\nat}
      }

  }}
  \bigskip

  And:

  \bigskip
  \noindent\makebox[\textwidth]{%
    \typerulederiv{$\text{\Cref{tr:t-let}}$}{
      \typerulederiv{$\text{\Cref{tr:w-t-delete}}$}{
        \src{\nat}\not=\src{\ptr} &

        \typechecks{w:\nat,z:\ptr,x:\nat,\hole{\cdot}}{delete\ z}{\nat}

      \typerulederiv{$\text{\Cref{tr:t-var}}$}{
        \src{\nat}\not=\src{\ptr} &
        \src{\nat}\not=\src{\ptr}

      \typechecks{w:\nat,z:\ptr,x:\nat,\hole{\cdot}\circ w:\nat,x:\nat,\hole{\cdot}}{delete\ z;w}{\nat}
    }}
  \bigskip

  Thus, it typechecks.\\[0.3cm]
  Now consider the other case:

  \bigskip
  \noindent\makebox[\textwidth]{%
    \typerulederiv{$\text{\Cref{tr:t-call}}$}{
      \typerulederiv{$\text{\Cref{tr:t-var}}$}{}{\typechecks{z:\nat,\Gamma_0}{foo}{\nat\to\nat}}&

    }}
  \bigskip

  Note that $\src{NoOwnedPtr\ z:\nat,\Gamma_{0}}$ holds by $\src{\nat}\not=\src{\ptr}$ and $\src{\nat\to\nat}\not=\src{\ptr}$

  Running the whole program via \Cref{tr:ws-e-prog} yields trace prefix\\$\src{\trace}=\src{Start;\comp}\cdot\src{Call\ !foo\ 42;\ctx}\cdot\src{Alloc\ \loc\ 42;\comp}\cdot\src{Get\ \loc\ 1337;\comp}\cdot\src{Dealloc\ \loc;\comp}\cdot\src{Ret\ ?1729;\comp}\cdot\src{End\ 1729;\ctx}$.
  We omit the precise derivation for brevity, but note that \thmref{lem:toplevel:progress} gives us the necessary execution.\\[0.3cm]
  Let $\delta_{ms}=\{\src{\loc}\mapsto\specificev{\loc}\}$, by \Cref{fig:w-events-project-to-specific}, $\tospecificevs[^*_{\delta_{ms}}]{\trace}=\specificev{Alloc\ \loc\ 42}\cdot\specificev{Use\ \loc\ 1337}\cdot\specificev{Dealloc\ \loc}$.\\[0.3cm]
  For \Cref{def:spatmemsafe}, note that $\specificev{Alloc\ \loc\ 42}\in\tospecificevs[^*_{\delta_{ms}}]{\trace}$ and $\specificev{Use\ \loc\ 1337}\in\tospecificevs[^*_{\delta_{ms}}]{\trace}$, but ${1337}\not<{42}$, hence $\nrsat{\src{\library_{comp}}}{{smsafe}}$.
\end{proof}

\begin{lemma}[Typed Linking Recomposition]\label{lem:typed:linking:recomposition}
  If
  \begin{assumptions}
    \item $\src{\library} = \src{\library_{1}\linker\library_{2}}$

    \item $\Gammas\vdash\src{\library_{2}}\operatorname{ok}$

  Then
  \begin{goals}

  \end{goals}
\end{lemma}

    \item[Case $\src{\library_{1}}=\src{F,\library_{1}'}$:]

  \end{description}
\end{proof}

\begin{lemma}[Typing Decomposition]\label{lem:ctxtyping:plugged}
  If
  \begin{assumptions}
    \item $\typechecks{\Gamma}{K[e]}{\type}$
  \end{assumptions}
  Then $\exists \src{\type_{e}}, \src{\Gamma_{\hole{\cdot}}}$,
  \begin{goals}
    \item $\ctxtypechecks{\Gamma}{K}{\type_{e}\to\type}{\Gamma_{\hole{\cdot}}}$
    \item $\typechecks{\Gamma_{\hole{\cdot}}}{e}{\type_{e}}$
  \end{goals}
\end{lemma}
\begin{proof}
  By induction on $\src{K}$.

  \case{tr:w-te-hole}
    $\src{K} = \src{\hole{\cdot}}$, so $\src{K[e]} = \src{e}$.
    We have $\typechecks{\Gamma}{e}{\type}$.
    By \Cref{tr:w-te-hole}, $\ctxtypechecks{\Gamma}{\hole{\cdot}}{\type\to\type}{\Gamma}$.
    Take $\src{\Gamma_{\hole{\cdot}}} = \Gammas$ and $\types_e = \types$.

  \case{tr:w-te-let}
    $\src{K} = \src{let\ x = K'\ in\ e'}$, so $\src{K[e]} = \src{let\ x = K'[e]\ in\ e'}$.
    By inversion of \Cref{tr:t-let}, $\Gammas \equiv \src{\Gamma_1} \circ \src{\Gamma_2}$ with:
    \begin{itemize}
      \item $\typechecks{\Gamma_1}{K'[e]}{\type_e'}$
      \item $\typechecks{x:\type_e', \Gamma_2}{e'}{\type}$
    \end{itemize}
    By IH on $\src{K'}$: $\ctxtypechecks{\Gamma_1}{K'}{\type_e\to\type_e'}{\Gamma_{\hole{\cdot}}}$ and $\typechecks{\Gamma_{\hole{\cdot}}}{e}{\type_e}$.
    By \Cref{tr:w-te-let}: $\ctxtypechecks{\Gamma_1\circ\Gamma_2}{let\ x=K'\ in\ e'}{\type_e\to\type}{\Gamma_{\hole{\cdot}}}$.

  Other cases follow similarly, propagating $\src{\Gamma_{\hole{\cdot}}}$ through the recursive structure.
\end{proof}
\begin{lemma}[Typing Composition]\label{lem:ctxtyping:compos}

  Other cases follow similarly.
\end{proof}

\begin{lemma}[Base Preservation]\label{lem:base-preservation}
  If
  \begin{assumptions}
    \item\label[ass]{asm:basepres:typs} $\typechecks{}{\src{\Omega}\triangleright \expr\gamma}{\src{\type}}$
    \item\label[ass]{asm:basepres:step} $\exprevals{\configs}{\expr\gamma}{\configs'}{\expr'\gamma'}{\event}$
    \item\label[ass]{asm:basepres:nocrash} $\configs'\not=\src{\lightning}$
  \end{assumptions}
  Then
  \begin{goals}
    \item $\typechecks{}{\src{\Omega'}\triangleright \expr'\gamma'}{\src{\type}}$
  \end{goals}.
\end{lemma}
\begin{proof}
  By induction on \Cref{asm:basepres:step}.
  In each case, we extract the typing context $\Gammas$ from $\src{\Omega}$ via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-oplus}
    $\src{\expr} = \src{n_1 \oplus n_2}$ and $\src{\expr'} = \src{n_3}$ where $\src{n_1} \oplus \src{n_2} = \src{n_3}$.
    Configuration unchanged: $\src{\Omega'} = \src{\Omega}$.
    By inversion of \Cref{asm:basepres:typs}, $\typechecks{\Gamma}{n_1 \oplus n_2}{\nat}$, so $\types = \nat$.
    By \Cref{tr:t-nat}, $\typechecks{\Gamma}{n_3}{\nat}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-get-in}
    $\src{\expr} = \src{x[n]}$ and $\src{\expr'} = \src{H^{\sandboxtag}(\loc+n)}$.
    Configuration unchanged: $\src{\Omega'} = \src{\Omega}$.
    By inversion, $\typechecks{\Gamma_1}{x}{\wptr}$ and $\typechecks{\Gamma_2}{n}{\nat}$ with $\Gammas = \src{\Gamma_1} \circ \src{\Gamma_2}$.
    Since heap values are natural numbers, $\typechecks{\Gamma}{H^{\sandboxtag}(\loc+n)}{\nat}$ by \Cref{tr:t-nat}.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-get-notin}
    Similar to above; result is $\src{1729}$ which has type $\nat$.

  \case{tr:w-e-set-in}
    $\src{\expr} = \src{x[n] \leftarrow \valueexpr}$ and $\src{\expr'} = \src{\valueexpr}$.
    By inversion, $\typechecks{\Gamma}{\valueexpr}{\nat}$.
    The heap is updated but the store $\Deltas$ is unchanged, so the typing context $\Gammas$ remains valid.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-set-notin}
    Similar to above.

  \case{tr:w-e-let-f}
    $\src{\expr} = \src{let\ x = \finalexprnoerr\ in\ \expr[_0]}$ and $\src{\expr'} = \src{\expr[_0][\finalexprnoerr/x]}$.
    By inversion of \Cref{tr:t-let}, $\Gammas = \src{\Gamma_1} \circ \src{\Gamma_2}$ with:
    \begin{itemize}
      \item $\typechecks{\Gamma_1}{\finalexprnoerr}{\type'}$
      \item $\typechecks{x:\type', \Gamma_2}{\expr[_0]}{\type}$
    \end{itemize}
    By \Cref{lem:substitution} with $\src{\Gamma_3} = \src{\Gamma_1} \circ \src{\Gamma_2}$:
    $\typechecks{\Gamma_1 \circ \Gamma_2}{\expr[_0][\finalexprnoerr/x]}{\type}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-delete}
    $\src{\expr} = \src{delete\ x}$ and $\src{\expr'} = \src{0}$.
    The store is updated to mark $\src{x}$ as poisoned.
    By \Cref{tr:t-poison-store}, poisoned locations do not contribute to $\Gammas$.
    Hence the new typing context $\src{\Gamma'}$ from the updated store satisfies $\typechecks{\Gamma'}{0}{\nat}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-let-new}
    $\src{\expr} = \src{let\ x = new\ n\ in\ \expr[_0]}$ and $\src{\expr'} = \src{\expr[_0][z/x]}$.
    A fresh location $\src{z}$ is allocated.
    By inversion of \Cref{tr:w-t-new}, $\Gammas = \src{\Gamma_1} \circ \src{\Gamma_2}$ with:
    \begin{itemize}
      \item $\typechecks{\Gamma_1}{n}{\nat}$
      \item $\typechecks{x:\ptr, \Gamma_2}{\expr[_0]}{\type}$
    \end{itemize}
    By \Cref{tr:t-ptr-store}, the new store yields context $\src{z:\ptr, \Gamma'}$.
    By $\alpha$-conversion (\Cref{lem:alpha-conv-typing}) and \Cref{lem:substitution}:
    $\typechecks{z:\ptr, \Gamma_1 \circ \Gamma_2}{\expr[_0][z/x]}{\type}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-if-true}
    $\src{\expr} = \src{ifz\ 0\ then\ \expr[_1]\ else\ \expr[_2]}$ and $\src{\expr'} = \src{\expr[_1]}$.
    By inversion of \Cref{tr:w-t-if}, $\typechecks{\Gamma}{\expr[_1]}{\type}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-if-false}
    Similar; by inversion $\typechecks{\Gamma}{\expr[_2]}{\type}$.

  \case{tr:w-e-abort}
    Excluded by \Cref{asm:basepres:nocrash} since $\src{\Omega'} = \src{\lightning}$.
\end{proof}

\begin{lemma}[Ctx Preservation]\label{lem:ctx-preservation}
  If
  \begin{assumptions}
    \item\label[ass]{asm:ctxpres:typs} $\typechecks{}{\src{\Omega}\triangleright \expr\gamma}{\src{\type}}$
    \item\label[ass]{asm:ctxpres:step} $\execs[]{\configs}{\expr\gamma}{\configs'}{\expr'\gamma'}{\event}$
  \end{assumptions}
  Then
  \begin{goals}
    \item $\typechecks{}{\src{\Omega'}\triangleright \expr'\gamma'}{\src{\type}}$
  \end{goals}.
\end{lemma}
\begin{proof}
  By induction on \Cref{asm:ctxpres:step}.

  \case{tr:w-e-ectx}
    We have $\src{\expr} = \src{K[\expr[_0]]}$, $\src{\expr'} = \src{K[\expr[_0]']}$, and $\exprevals{\Omega}{\expr[_0]}{\Omega'}{\expr[_0]'}{\event}$.

    By \Cref{lem:rtctxtyping:plugged} on \Cref{asm:ctxpres:typs}:
    \begin{itemize}
      \item $\ctxtypechecks{}{\Omega \triangleright K}{\type_e \to \type}{\Gamma_{\hole{\cdot}}}$
      \item $\typechecks{\Gamma_{\hole{\cdot}}}{\expr[_0]}{\type_e}$
    \end{itemize}

    If $\src{\Omega'} = \src{\lightning}$, then the goal holds vacuously (crashed configurations are well-typed by convention).

    Otherwise, by \Cref{lem:base-preservation}:
    $\typechecks{}{\Omega' \triangleright \expr[_0]'}{\type_e}$.

    The evaluation context $\src{K}$ is unchanged, and the portion of the context $\src{\Gamma_{\hole{\cdot}}}$ that flows to the hole may be updated according to the store changes in $\src{\Omega'}$.
    By inspection of the base evaluation rules (\Cref{fig:w-expr-prim-eval}), store updates preserve the typing derivation structure.

    By \Cref{lem:rtctxtyping:compos}:
    $\typechecks{}{\Omega' \triangleright K[\expr[_0]']}{\type}$.

  \case{tr:w-e-ectx-stuck}
    We have $\exprevals{\Omega}{\expr[_0]}{\Omega'}{stuck}{\lightning}$.
    Then $\src{\expr'} = \src{stuck}$ and $\src{\Omega'}$ represents a crashed state.
    Crashed configurations are well-typed by convention.

  \case{tr:w-e-ectx-call-main}
    We have $\src{\expr} = \src{K[call\ main\ \valueexpr]}$ and $\src{\expr'} = \src{\expr[_{\text{body}}][\valueexpr/x]}$.
    The library contains $\src{let\ main\ x:\type_\lambda := \expr[_{\text{body}}]}$.

    By \Cref{lem:rtctxtyping:plugged}:
    \begin{itemize}
      \item $\ctxtypechecks{}{\Omega \triangleright K}{\type_e \to \type}{\Gamma_{\hole{\cdot}}}$
      \item $\typechecks{\Gamma_{\hole{\cdot}}}{call\ main\ \valueexpr}{\type_e}$
    \end{itemize}

    By inversion of \Cref{tr:t-call}, $\types_e = \types_{\text{ret}}$ where $\src{main} : \src{\type_{\text{arg}} \to \type_{\text{ret}}} \in \src{\Gamma_0}$.
    By the well-formedness of the library (\Cref{tr:w-t-wprog}): $\typechecks{x:\type_{\text{arg}}, \Gamma_0}{\expr[_{\text{body}}]}{\type_{\text{ret}}}$.

    By \Cref{lem:substitution}: $\typechecks{\Gamma_0}{\expr[_{\text{body}}][\valueexpr/x]}{\type_{\text{ret}}}$.
    The continuation stack is updated to $(K;main), \src{\hole{\cdot}}$.

    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-ectx-call-notsame}
    Similar to above. The function $\src{foo}$ is in a different compartment.
    By library well-formedness and \Cref{lem:substitution}, the body is well-typed.
    The continuation stack and sandbox tag are updated accordingly.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-ectx-call-same}
    Similar to above. The function $\src{foo}$ is in the same compartment.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-ectx-ret-main}
    We have $\src{\expr} = \src{K'[return\ \valueexpr]}$ with continuation stack $\src{\hole{\cdot}^{main}, \hole{\cdot}}$.
    The result is $\src{\expr'} = \src{\valueexpr}$.
    By inversion, $\typechecks{\Gamma}{\valueexpr}{\type_{\text{ret}}}$.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.

  \case{tr:w-e-ectx-ret-notsame}
    We have continuation stack $\src{(K;foo), \kontstack}$ where $\src{foo}$ is in a different compartment.
    The return restores the context $\src{K}$ with the value $\src{\valueexpr}$ plugged in.
    By inversion of the call typing, $\src{K}$ expects a value of type $\types_{\text{ret}}$.
    By \Cref{lem:rtctxtyping:compos}: $\typechecks{}{\Omega' \triangleright K[\valueexpr]}{\type}$.

  \case{tr:w-e-ectx-ret-same}
    Similar to above; the function is in the same compartment.
    Reconstruct via \Cref{tr:w-t-prog-runtime}.
\end{proof}

\begin{lemma}[N-Steps Preservation]\label{lem:nsteps-preservation}
  If
  \begin{assumptions}
    \item\label[ass]{asm:npres:typs} $\typechecks{}{\src{\Omega}\triangleright \expr\gamma}{\src{\type}}$
    \item\label[ass]{asm:npres:step} $\execsn{\configs}{\expr\gamma}{\configs'}{\expr'\gamma'}{\trace}$
  \end{assumptions}
  Then
  \begin{goals}
    \item $\typechecks{}{\src{\Omega'}\triangleright \expr'\gamma'}{\src{\type}}$
  \end{goals}.
\end{lemma}
\begin{proof}
  By induction on $n$ in \Cref{asm:npres:step}.

  \case{tr:wsn-e-refl} ($n = 0$)
    We have $\execsn[^0]{\Omega}{\expr}{\Omega}{\expr}{\emptyevent}$.
    The configuration and expression are unchanged: $\src{\Omega'} = \src{\Omega}$ and $\src{\expr'} = \src{\expr}$.
    The goal follows immediately from \Cref{asm:npres:typs}.

  \case{tr:wsn-e-trans-important} ($n = k + 1$)
    We have:
    \begin{itemize}
      \item $\execs[]{\Omega}{\expr}{\Omega_1}{\expr[_1]}{\event}$ with $\src{\event} \neq \src{\emptyevent}$
      \item $\execsn[^k]{\Omega_1}{\expr[_1]}{\Omega'}{\expr'}{\trace}$
    \end{itemize}

    By \Cref{lem:ctx-preservation} applied to the first step:
    $\typechecks{}{\Omega_1 \triangleright \expr[_1]}{\type}$.

    By the induction hypothesis on $k$:
    $\typechecks{}{\Omega' \triangleright \expr'}{\type}$.

  \case{tr:wsn-e-trans-unimportant} ($n = k + 1$)
    We have:
    \begin{itemize}
      \item $\execs[]{\Omega}{\expr}{\Omega_1}{\expr[_1]}{\emptyevent}$
      \item $\execsn[^k]{\Omega_1}{\expr[_1]}{\Omega'}{\expr'}{\trace}$
    \end{itemize}

    By \Cref{lem:ctx-preservation} applied to the first step:
    $\typechecks{}{\Omega_1 \triangleright \expr[_1]}{\type}$.

    By the induction hypothesis on $k$:
    $\typechecks{}{\Omega' \triangleright \expr'}{\type}$.
\end{proof}

\begin{lemma}[Progress]\label{lem:progress}
  If
  \begin{assumptions}
    \item\label[ass]{asm:progress:typs} $\typechecks{}{\src{\Omega}\triangleright \expr\gamma}{\src{\type}}$
  \end{assumptions}
  Then $\exists \src{\Omega'}\ \src{\finalexpr}\ \src{\trace}$,
  \begin{goals}
  \item $\execs{\configs}{\expr}{\configs'}{\finalexpr}{\trace}$
  \end{goals}.
\end{lemma}
\begin{proof}
  By nested induction: outer induction on the structure of $\src{\expr}$, inner induction on evaluation contexts when needed.
  We extract the typing context $\Gammas$ from $\src{\Omega}$ via \Cref{tr:w-t-prog-runtime}.

  \begin{description}
  \item[Case $\src{\expr} = \src{\finalexprnoerr}$] (values or variables).
    The expression is already a final result.
    By \Cref{tr:ws-e-refl}: $\execs{\Omega}{\finalexprnoerr}{\Omega}{\finalexprnoerr}{\hole{\cdot}}$.

  \item[Case $\src{\expr} = \src{n_1 \oplus n_2}$] (both operands are values).
    By \Cref{tr:w-e-oplus}: $\exprevals{\Omega}{n_1 \oplus n_2}{\Omega}{n_3}{\emptyevent}$ where $\src{n_3} = \src{n_1} \oplus \src{n_2}$.
    By \Cref{tr:w-e-ectx} with $\src{K} = \src{\hole{\cdot}}$: $\execs[]{\Omega}{n_1 \oplus n_2}{\Omega}{n_3}{\emptyevent}$.
    By \Cref{tr:ws-e-refl,tr:ws-e-trans-unimportant}: $\execs{\Omega}{n_1 \oplus n_2}{\Omega}{n_3}{\hole{\cdot}}$.

  \item[Case $\src{\expr} = \src{\expr[_1] \oplus \expr[_2]}$] (at least one non-value).
    \begin{itemize}
      \item If $\src{\expr[_1]}$ is not a value:
        By IH on $\src{\expr[_1]}$: $\execs{\Omega}{\expr[_1]}{\Omega'}{\finalexpr[_1]}{\trace[_1]}$.
        The evaluation context is $\src{K} = \src{\hole{\cdot} \oplus \expr[_2]}$.
        By lifting through the context: $\execs{\Omega}{\expr[_1] \oplus \expr[_2]}{\Omega''}{\finalexpr}{\trace}$.
      \item If $\src{\expr[_1]} = \src{\valueexpr[_1]}$ is a value, $\src{\expr[_2]}$ is not:
        By IH on $\src{\expr[_2]}$: $\execs{\Omega}{\expr[_2]}{\Omega'}{\finalexpr[_2]}{\trace[_2]}$.
        The evaluation context is $\src{K} = \src{\valueexpr[_1] \oplus \hole{\cdot}}$.
        By lifting through the context: $\execs{\Omega}{\valueexpr[_1] \oplus \expr[_2]}{\Omega''}{\finalexpr}{\trace}$.
    \end{itemize}

  \item[Case $\src{\expr} = \src{x[n]}$] (index is a value).
    By well-typedness, $\src{x} : \src{\wptr} \in \Gammas$.
    By \Cref{tr:w-t-prog-runtime}, $\src{x}$ is bound in $\src{\Omega.\Delta}$.
    Either $\src{\loc + n} \in \operatorname{dom}(\src{H^{\sandboxtag}})$ or not:
    \begin{itemize}
      \item If in domain: by \Cref{tr:w-e-get-in}, $\exprevals{\Omega}{x[n]}{\Omega}{H^{\sandboxtag}(\loc+n)}{\ldots}$.
      \item If not in domain: by \Cref{tr:w-e-get-notin}, $\exprevals{\Omega}{x[n]}{\Omega}{1729}{\ldots}$.
    \end{itemize}
    Lift to multi-step via \Cref{tr:w-e-ectx,tr:ws-e-trans-important,tr:ws-e-refl}.

  \item[Case $\src{\expr} = \src{x[\expr[_1]]}$] ($\src{\expr[_1]}$ not a value).
    By IH on $\src{\expr[_1]}$: $\execs{\Omega}{\expr[_1]}{\Omega'}{\finalexpr[_1]}{\trace[_1]}$.
    Lift through context $\src{K} = \src{x[\hole{\cdot}]}$.

  \item[Case $\src{\expr} = \src{x[n] \leftarrow \valueexpr}$] (both index and value are values).
    Similar to get case. By \Cref{tr:w-e-set-in} or \Cref{tr:w-e-set-notin}.

  \item[Case $\src{\expr} = \src{x[\expr[_1]] \leftarrow \expr[_2]}$] (at least one non-value).
    By IH on the leftmost non-value sub-expression, lift through appropriate context.

  \item[Case $\src{\expr} = \src{let\ x = \finalexprnoerr\ in\ \expr[_0]}$.]
    By \Cref{tr:w-e-let-f}: $\exprevals{\Omega}{let\ x = \finalexprnoerr\ in\ \expr[_0]}{\Omega}{\expr[_0][\finalexprnoerr/x]}{\emptyevent}$.
    Lift to multi-step.

  \item[Case $\src{\expr} = \src{let\ x = \expr[_1]\ in\ \expr[_2]}$] ($\src{\expr[_1]}$ not final).
    By IH on $\src{\expr[_1]}$: $\execs{\Omega}{\expr[_1]}{\Omega'}{\finalexpr[_1]}{\trace[_1]}$.
    Lift through context $\src{K} = \src{let\ x = \hole{\cdot}\ in\ \expr[_2]}$.

  \item[Case $\src{\expr} = \src{let\ x = new\ n\ in\ \expr[_0]}$.]
    By \Cref{tr:w-e-let-new}: a fresh location is allocated.
    $\exprevals{\Omega}{let\ x = new\ n\ in\ \expr[_0]}{\Omega'}{\expr[_0][z/x]}{\ldots}$.
    Lift to multi-step.

  \item[Case $\src{\expr} = \src{let\ x = new\ \expr[_1]\ in\ \expr[_0]}$] ($\src{\expr[_1]}$ not a value).
    By IH on $\src{\expr[_1]}$, lift through context $\src{K} = \src{let\ x = new\ \hole{\cdot}\ in\ \expr[_0]}$.

  \item[Case $\src{\expr} = \src{delete\ x}$.]
    By \Cref{tr:w-e-delete}: $\exprevals{\Omega}{delete\ x}{\Omega'}{0}{\ldots}$.
    Lift to multi-step.

  \item[Case $\src{\expr} = \src{ifz\ 0\ then\ \expr[_1]\ else\ \expr[_2]}$.]
    By \Cref{tr:w-e-if-true}: $\exprevals{\Omega}{ifz\ 0\ then\ \expr[_1]\ else\ \expr[_2]}{\Omega}{\expr[_1]}{\emptyevent}$.
    By IH on $\src{\expr[_1]}$: $\execs{\Omega}{\expr[_1]}{\Omega'}{\finalexpr}{\trace}$.
    Combine steps.

  \item[Case $\src{\expr} = \src{ifz\ S(n)\ then\ \expr[_1]\ else\ \expr[_2]}$.]
    By \Cref{tr:w-e-if-false}: step to $\src{\expr[_2]}$.
    By IH on $\src{\expr[_2]}$: complete evaluation.

  \item[Case $\src{\expr} = \src{ifz\ \expr[_0]\ then\ \expr[_1]\ else\ \expr[_2]}$] ($\src{\expr[_0]}$ not a value).
    By IH on $\src{\expr[_0]}$, lift through context $\src{K} = \src{ifz\ \hole{\cdot}\ then\ \expr[_1]\ else\ \expr[_2]}$.

  \item[Case $\src{\expr} = \src{call\ foo\ \valueexpr}$.]
    By well-typedness, $\src{foo}$ is defined in $\src{\Omega.\library}$.
    By \Cref{tr:w-e-ectx-call-main}, \Cref{tr:w-e-ectx-call-notsame}, or \Cref{tr:w-e-ectx-call-same}:
    the call steps to the function body with argument substituted.
    By IH on the function body: complete evaluation.

  \item[Case $\src{\expr} = \src{call\ foo\ \expr[_0]}$] ($\src{\expr[_0]}$ not a value).
    By IH on $\src{\expr[_0]}$, lift through context $\src{K} = \src{call\ foo\ \hole{\cdot}}$.

  \item[Case $\src{\expr} = \src{return\ \valueexpr}$.]
    By \Cref{tr:w-e-ectx-ret-main}, \Cref{tr:w-e-ectx-ret-notsame}, or \Cref{tr:w-e-ectx-ret-same}:
    the return pops the continuation stack and resumes the caller.
    By IH: complete evaluation of the resumed computation.

  \item[Case $\src{\expr} = \src{return\ \expr[_0]}$] ($\src{\expr[_0]}$ not a value).
    By IH on $\src{\expr[_0]}$, lift through context $\src{K} = \src{return\ \hole{\cdot}}$.

  \item[Case $\src{\expr} = \src{abort()}$.]
    By \Cref{tr:w-e-abort}: $\exprevals{\Omega}{abort()}{\lightning}{stuck}{\lightning}$.
    Lift to multi-step with crash.
  \end{description}
\end{proof}

\begin{lemma}[Top-Level Progress]\label{lem:toplevel:progress}
  If
  \begin{assumptions}
    \item\label[ass]{asm:toplevel:typs} $\vdash\src{prog\ \library_{ctx}\ \library_{comp}}\dashv\src{\library},\src{\commlib}$
  \end{assumptions}

  \begin{goals}
  \item $\wexecs{prog\ \library_{ctx}\ \library_{comp}}{\Omega}{\finalexpr}{\trace}$
  \item $\src{\Omega.\library}=\src{\library}$
  \item $\src{\Omega.\commlib}=\src{\commlib}$
  \end{goals}.
\end{lemma}
\begin{proof}
  By \Cref{asm:toplevel:typs} and inversion of \Cref{tr:w-t-wprog}:
  \begin{itemize}
    \item $\src{\library} = \src{\library_{ctx} \linker \library_{comp}}$
    \item $\src{\library} \downarrow = \src{\Gamma_0}$
    \item $\src{main} \in \operatorname{dom}(\src{\Gamma_0})$
    \item $\typechecks{\Gamma_0}{call\ main\ 0}{\nat \to \nat}$
  \end{itemize}

  Construct the initial configuration:
  $\src{\Omega_0} = \src{\commlib; \library; \hole{\cdot}; \comp; \hole{\cdot}; \hole{\cdot}; \hole{\cdot}}$.

  By \Cref{tr:t-empty-store}: $\typechecks{\library}{\hole{\cdot}}{\Gamma_0}$.
  By \Cref{tr:w-t-prog-runtime}: $\typechecks{}{\Omega_0 \triangleright call\ main\ 0}{\nat \to \nat}$.

  By \Cref{lem:progress}: $\exists \src{\Omega'}\ \src{\finalexpr}\ \src{\trace}$ such that
  $\execs{\Omega_0}{call\ main\ 0}{\Omega'}{\finalexpr}{\trace}$.

  By \Cref{tr:ws-e-prog} or \Cref{tr:ws-e-prog-crash}:
  $\wexecs{prog\ \library_{ctx}\ \library_{comp}}{\Omega'}{\finalexpr}{\trace'}$.

  The library and component library are preserved throughout execution:
  $\src{\Omega'.\library} = \src{\library}$ and $\src{\Omega'.\commlib} = \src{\commlib}$.
\end{proof}

    Note that $\tospecificevs[_{\delta_{\text{MS}}}]{Ret\ \comm\ \valueexpr;\sandboxtag}=\specificev{\varepsilon}$ for any $\delta_{\text{MS}}$ and $\src{\sandboxtag}$ whatsoever.
    So, instantiate $\absevent=\bm{\varepsilon}$, $\delta_{\text{MS}}'=\delta_{\text{MS}}$, and $\tmsmonitor[']=\tmsmonitor$.
    All goals follow easily.
  \end{description}
\end{proof}

\begin{lemma}[Steps \gls{tms} via Monitor]\label{lem:generaltms:monitor}
  If
  \begin{assumptions}
    \item\label[ass]{asm:stepstmsmonitor:typing} $\typechecks{}{\src{\Omega}\triangleright e\gamma}{\src{\type}}$
    \item\label[ass]{asm:stepstmsmonitor:exec} $\execs{\configs}{e\gamma}{\configs'}{e'\gamma'}{\trace}$
    \item\label[ass]{asm:stepstmsmonitor:storeagree} $\storeagreetms{\tmsmonitor}{\src{\Omega}.\src{\Delta}}$
  \end{assumptions}
  Then $\exists \abstrace\ \delta_{\text{MS}}'\ \tmsmonitor['],$
  \begin{goals}
    \item\label[goal]{goal:stepstmsmonitor:delta} $\delta_{\text{MS}}\subseteq\delta_{\text{MS}}'$
    \item\label[goal]{goal:stepstmsmonitor:traceagree} $\tmstraceagree[^{*}]{\tospecificevs[_{\delta_{\text{MS}}'}]{\trace}}{\abstrace}$
    \item\label[goal]{goal:stepstmsmonitor:monitorstep} $\monitorcheck{\tmsmonitor}{\tmsmonitor[']}{\abstrace}$
    \item\label[goal]{goal:stepstmsmonitor:storeagree} $\storeagreetms[\delta_{\text{MS}}']{\tmsmonitor[']}{\src{\Omega'}.\src{\Delta}}$
  \end{goals}.
\end{lemma}
\begin{proof}
  Induction on \Cref{asm:stepstmsmonitor:exec}.
  \begin{description}
    \item[$\src{\configs'}=\configs$, $\src{e'}=\src{e}$, and $\src{\trace}=\src{\hole{\cdot}}$:]
          This is the base case where no execution step occurs.
          Instantiate $\abstrace=\hole{\cdot}$, $\delta_{\text{MS}}'=\delta_{\text{MS}}$, and $\tmsmonitor[']=\tmsmonitor$.
          \Cref{goal:stepstmsmonitor:delta} holds since $\delta_{\text{MS}}\subseteq\delta_{\text{MS}}$ by reflexivity of subset.
          \Cref{goal:stepstmsmonitor:traceagree} holds since $\tospecificevs[_{\delta_{\text{MS}}}]{\hole{\cdot}}=\hole{\cdot}$ by definition of trace projection on empty traces, and then $\tmstraceagree[^{*}]{\hole{\cdot}}{\hole{\cdot}}$ follows by \Cref{tr:tms-refl-auth}.
          \Cref{goal:stepstmsmonitor:monitorstep} holds by \Cref{tr:tms-refl}, noting the monitor state is unchanged.
          \Cref{goal:stepstmsmonitor:storeagree} follows directly from \Cref{asm:stepstmsmonitor:storeagree}.

    \item[Induction step:]
          The inductive hypothesis is as follows.
          If
          \begin{passumptions}{A}
            \item $\typechecks{}{\src{\Omega_{\IH}}\triangleright e_{\IH}\gamma_{\IH}}{\src{\type}}$
            \item $\storeagreetms[\delta^{0}_{\text{MS}}]{\tmsmonitor[_{\IH}]}{\src{\Omega_{\IH}}.\src{\Delta}}$
          \end{passumptions}
          then $\exists \abstrace_{\IH}\ \delta_{\text{MS}}^{\IH}\ \tmsmonitor[_{\IH}'],$
          \begin{passumptions}{\IH}
            \item $\delta_{\text{MS}}^{0}\subseteq\delta_{\text{MS}}^{\IH}$
            \item $\tmstraceagree{\tospecificevs[_{\delta^{\IH}_{\text{MS}}}]{\trace}}{\abstrace_{\IH}}$
            \item $\monitorcheck{\tmsmonitor_{\IH}}{\tmsmonitor[_{\IH}']}{\abstrace_{\IH}}$
            \item $\storeagreetms[\delta^{\IH}_{\text{MS}}]{\tmsmonitor[_{\IH}']}{\src{\Omega'}.\src{\Delta}}$
          \end{passumptions}
          Note that above, anything not bound by $\exists$ is universally quantified, besides $\src{\type}$ and $\src{\Omega'}$.
          While our remaining context (excluding the inductive hypothesis) is:
          If
          \begin{passumptions}{H}
            \item\label[ass]{asm:stepstmsmonitor:case0:step} $\execs[]{\configs}{e\gamma}{\configs_{0}}{e_{0}\gamma_{0}}{\event[_0]}$
            \item\label[ass]{asm:stepstmsmonitor:case0:steps} $\execs{\configs_{0}}{e_{0}\gamma_{0}}{\configs'}{e'\gamma'}{\trace}$
            \item\label[ass]{asm:stepstmsmonitor:case0:checks} $\typechecks{}{\src{\Omega}\triangleright e\gamma}{\src{\type}}$
            \item\label[ass]{asm:stepstmsmonitor:case0:storeagree} $\storeagreetms{\tmsmonitor}{\src{\Omega}.\src{\Delta}}$
          \end{passumptions}
          Then $\exists \abstrace\ \delta_{\text{MS}}'\ \tmsmonitor['],$
          \begin{goals}
            \item\label[goal]{goal:stepstmsmonitor:final:delta} $\delta_{\text{MS}}\subseteq\delta_{\text{MS}}'$
            \item\label[goal]{goal:stepstmsmonitor:final:traceagree} $\tmstraceagree{\tospecificevs[_{\delta_{\text{MS}}'}]{\event[_{0}]\cdot\trace}}{\abstrace}$
            \item\label[goal]{goal:stepstmsmonitor:final:monitorstep} $\monitorcheck{\tmsmonitor}{\tmsmonitor[']}{\abstrace}$
            \item\label[goal]{goal:stepstmsmonitor:final:storeagree} $\storeagreetms[\delta_{\text{MS}}']{\tmsmonitor[']}{\src{\Omega'}.\src{\Delta}}$
          \end{goals}
          Use \Thmref{lem:ctxsteptmsmonitor} with \Cref{asm:stepstmsmonitor:case0:step,asm:stepstmsmonitor:case0:checks,asm:stepstmsmonitor:case0:storeagree}, obtaining:
          \begin{passumptions}{F}
            \item\label[ass]{asm:stepstmsmonitor:deltasubset0} $\delta_{\text{MS}}\subseteq\delta_{\text{MS}}^{0}$
            \item\label[ass]{asm:stepstmsmonitor:traceagree0} $\tmstraceagree{\tospecificevs[_{\delta^{0}_{\text{MS}}}]{\event[_{0}]}}{\absevent_{0}}$
            \item\label[ass]{asm:stepstmsmonitor:monitorcheck0} $\monitorcheck[]{\tmsmonitor}{\tmsmonitor[_{0}]}{\absevent_{0}}$
            \item\label[ass]{asm:stepstmsmonitor:case0:storeagree1} $\storeagreetms[\delta^{0}_{\text{MS}}]{\tmsmonitor[_{0}]}{\src{\Omega_{0}}.\src{\Delta}}$
          \end{passumptions}
          Discharge the inductive hypothesis with \Cref{asm:stepstmsmonitor:case0:storeagree1} and the result of applying \Thmref{lem:ctx-preservation} using \Cref{asm:stepstmsmonitor:case0:step,asm:stepstmsmonitor:case0:checks}:
          \begin{passumptions}{G}
            \item\label[ass]{asm:stepstmsmonitor:deltasubset1} $\delta_{\text{MS}}^{0}\subseteq\delta_{\text{MS}}^{\IH}$
            \item\label[ass]{asm:stepstmsmonitor:traceagree1} $\tmstraceagree{\tospecificevs[_{\delta^{\IH}_{\text{MS}}}]{\trace}}{\abstrace_{\IH}}$
            \item\label[ass]{asm:stepstmsmonitor:monitorcheck1} $\monitorcheck{\tmsmonitor_{0}}{\tmsmonitor[^{\IH}]}{\abstrace_{\IH}}$
            \item\label[ass]{asm:stepstmsmonitor:case0:storeagree2} $\storeagreetms[\delta^{\IH}]{\tmsmonitor[^{\IH}]}{\src{\Omega'}.\src{\Delta}}$
          \end{passumptions}

          We solve our goals by instantiating $\abstrace=\absevent\cdot\abstrace_{\IH}$, $\delta_{\text{MS}}'=\delta_{\text{MS}}^{\IH}$, and $\tmsmonitor[']=\tmsmonitor[^{\IH}]$.
          \Cref{goal:stepstmsmonitor:final:delta} follows by transitivity using \Cref{asm:stepstmsmonitor:deltasubset0,asm:stepstmsmonitor:deltasubset1}.

          Similarily for \Cref{goal:stepstmsmonitor:final:traceagree} using \Cref{asm:stepstmsmonitor:traceagree0,asm:stepstmsmonitor:traceagree1} and \Thmref{lem:filter-weaken}.

          \Cref{goal:stepstmsmonitor:final:monitorstep} also by transitivity using \Cref{asm:stepstmsmonitor:monitorcheck0,asm:stepstmsmonitor:monitorcheck1}.

          \Cref{goal:stepstmsmonitor:final:storeagree}, again, by transitivity using \Cref{asm:stepstmsmonitor:case0:storeagree1,asm:stepstmsmonitor:case0:storeagree2}, making use of \Thmref{lem:store-agree-weaken}.
  \end{description}
\end{proof}
